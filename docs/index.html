02
The Posture

Four apps. The same three promises, four times over.

01
Local-first
Nothing leaves your machine by default.
+
Every one of the four runs as a process you own — on your laptop, your LAN, your own relay. No SaaS tier sits above them, no telemetry phones home, no account is required to run anything. Your data is plain files on your own disk, and it stays there.
02
MIT, all of it
Read the source. Fork it. Keep it.
+
All four are MIT-licensed and fully open. No paid tier gates a feature, no proprietary store locks in your notes. Memcon's vault, Thymeline's entries, and Invpart's ledger are all Obsidian-compatible markdown — walk away with everything, any time.
03
Engineer-shaped
For code + hardware + parts + threads, together.
+
Not generic project management. Each tool fits the specific shape of building a real thing: debugging a hard problem with an AI, handing a teammate a snippet without leaving a trace, logging a small win before you forget it, knowing whether the servo has actually arrived. Sharp wedges, not a Swiss-army suite.
03
Origin

All four were built to make one robot.

BARQ is an autonomous quadruped robot, built by one engineer. Notion, Linear, Slack, and warehouse inventory tools all fell short for the specific shape of the work — code and hardware and parts and long debugging threads, all at once.

So instead of bending the work to the tools, the tools got built to fit the work. Engram is what came out: four sharp instruments, each solving one real friction, then generalised so they work for any project with the same shape.

04
Mix & Match

Use one. Use four. Never more than you need.

There is no "install Engram." You install the one that solves the problem in front of you.

Engram is an umbrella, not a bundle. The four share a posture and a design language — not a database. Each installs on its own path and runs without the others.

  • No shared backbone to stand up first
  • No account that unlocks "the suite"
  • Discover the other three when you need them
Your problem
Claude starts cold every time.
Memcon gives Claude a local memory layer over MCP. It auto-queries your past work before answering and writes the solution back after you solve it — no copy-paste, no cloud, no second app to open.
Install Memcon↗ Just this one. The other three stay on the shelf.
Your problem
Every channel keeps a copy forever.
Memshare hands a teammate code, files, or a quick comment over an end-to-end encrypted room. The relay is genuinely dumb — it only ever sees ciphertext. Nothing is logged, nothing persists past the session.
Install Memshare↗ Just this one. The other three stay on the shelf.
Your problem
By V2 you've forgotten the V1 fix.
Thymeline is a build journal with one quick-entry bar that auto-detects type from whatever you paste or drop. Thought to logged in one keystroke — so the encoder-drift fix you shipped in V1 is still there when V2 breaks the same way.
Get Thymeline↗ Just this one. The other three stay on the shelf.
Your problem
"Can I start today, or am I waiting on a servo?"
Invpart is a parts ledger where a four-state status flag on every part drives every total at once — spent, planned, remaining, pending. One glance answers whether the subassembly can start. Markdown source, Obsidian-editable.
Get Invpart↗ Just this one. The other three stay on the shelf.
