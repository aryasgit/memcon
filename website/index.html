The Loop

Claude asks. Memcon answers. It writes the result back. Forever.

No copy-paste. No manual note-taking. The vault grows itself while you work.

1
You
"Why is the RR wrist servo overheating again?"
2
Claude · MCP call
Claude reaches for Memcon to look up your past work on this symptom.
memcon_query("RR wrist servo overheating")
3
Memcon · semantic search
Top-5 chunks from your vault. Not the whole memory — only what semantically matches the symptoms.
→ 5 chunks: rr_wrist_overheating, i2c_oserror, gait_log_03 …
4
Claude · grounded answer
"Last time, the cause was vibration-loosened wiring causing a power brownout during backward gait. The fix was tightening connectors and redistributing static load on the RR leg."
5
Claude · MCP write
You confirm the new fix. Claude writes it back without you typing anything.
memcon_write_debug(title, symptom, cause, fix, subsystem="servo")
6
Memcon · persisted
New markdown note lands in vault/debugging/. Obsidian sees it instantly. Watcher re-ingests it. Searchable next session.
Why this matters

Not RAG. Not a chatbot. A backend brain.

Most "AI memory" solutions are cloud services that read your code. Memcon is a process that lives next to Claude on your machine.

fig.01 / local

Your machine. Your memory.

Qdrant in Docker, sentence-transformers locally, Ollama for the LLM. Nothing in this stack touches the cloud. Your engineering history is yours.

fig.02 / hashmap

Keyed by meaning, not strings.

Ask about a "weird motor error" — get back the note that talks about "servo torque loss." Embeddings catch what keyword search misses.

fig.03 / mcp

Claude does the work.

Nine MCP tools. Claude decides when to query and when to write. You just work. The reflex is "ask Claude" — Memcon disappears into the background.

fig.04 / obsidian

Markdown on disk.

Every "memory" is a real .md file in vault/. Open the folder in Obsidian. Edit, link, browse. No proprietary database to migrate out of.

fig.05 / free

$0 forever.

No API keys. No subscriptions. No vendor lock-in. The recurring cost is exactly zero — electricity not included.

fig.06 / open

MIT licensed.

Fork it, modify it, ship it inside your own product. Built for engineers who build hard things.

In practice

Six lines of config. That's the integration.

After install, paste this into Claude Desktop's MCP config and restart.

~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "memcon": { "command": "/Users/you/memcon/.venv/bin/python3", "args": ["-m", "memcon_mcp.server"], "cwd": "/Users/you/memcon" } } }
